# astrbot_plugin_mimo_tts

对接小米 MiMo API 的 AstrBot 语音合成插件。支持 V2.5-TTS 系列三种模型、预置音色、音色设计与音色克隆。

## 功能

- **自动模式** — LLM 回复时按概率自动转为语音发送（可配置概率/长度上限/LLM结果过滤/去重/空文本过滤）
- **手动指令** — `/mimo说 <内容>` 直接合成语音
- **LLM 工具** — Agent 可主动调用 `mimo_tts(message, style)` 输出语音，工具描述包含详细的风格控制示例
- **可配置工具调用指引** — 在配置页面自定义 LLM 调用 mimo_tts 工具的频率和场景
- **统一发送顺序** — 三个入口（自动/手动/工具）均先发音频再发文本，避免空消息问题

## 与 AstrBot 内置 MiMo TTS 的对比

AstrBot 自身在「模型提供商」中已支持接入 MiMo TTS（提供商名 `mimo_tts_api`），但仅提供最基础的语音合成引擎。本插件在其之上增加了多层能力：

| 能力 | 内置提供商 | 本插件 |
|------|-----------|--------|
| 基础语音合成 | ✅ | ✅ |
| V2.5-TTS 精品模型 | ❌ | ✅ |
| VoiceDesign（文字设计音色） | ❌ | ✅ |
| VoiceClone（音频克隆音色） | ❌ | ✅ |
| 9 种预置精品音色 | ❌ | ✅ |
| 自然语言风格指令 | ❌ | ✅ |
| 音频标签细粒度控制 | ❌ | ✅ |
| 唱歌模式 | ❌ | ✅ |
| 音频文件缓存 | ❌ | ✅ |
| PCM16 自动转 WAV | ❌ | ✅ |
| 自动模式（概率触发 TTS） | ❌ | ✅（按概率/长度/LLM过滤/去重/空文本过滤） |
| 手动指令（命令式触发） | ❌ | ✅ |
| LLM 工具调用（`mimo_tts`） | ❌ | ✅（含详细参数描述和可配置调用指引） |
| WebUI 完整配置面板 | ❌ | ✅（含滑条/下拉/模板列表） |

## 支持的模型

> **注意**：`mimo-v2-tts` 已于 2026 年 6 月下线，不再提供服务。如仍需兼容旧配置，插件会将其自动路由至 V2.5 系列。

| 模型 | 音色 | 唱歌 | 标签格式 | 自然语言指令 |
|------|------|------|---------|------------|
| `mimo-v2.5-tts` | **9 种精品**（推荐） | ✅ `(唱歌)` | `(风格)文本` + `[标签]` | ✅（含导演模式） |
| `mimo-v2.5-tts-voicedesign` | **文字描述生成**（无需参考音频） | ❌ | `(风格)文本` + `[标签]` | ✅（音色描述） |

## 默认配置

插件默认使用 `mimo-v2.5-tts` 模型 + `mimo_default` 音色，开箱即用。

## 部署

### 1. 获取 API Key

前往 [小米 MiMo 开放平台](https://platform.xiaomimimo.com) 注册并创建 API Key。

### 2. 安装插件

在 AstrBot WebUI 的插件市场安装此插件，或手动放入 `data/plugins/` 目录。

### 3. 配置

WebUI 插件页中：

1. **总开关** — 打开 `enabled`
2. **API 配置** — 填入 `api_key`
3. **模型选择** — 按需求选模型（默认 V2.5）
4. **音色选择** — 从预置列表中选一个（默认 `mimo_default`）
5. **默认风格** — 可选，如 `语气自然，像一个会聊天的私人助手`

### 4. 依赖

```
pip install -U openai
```

## 配置项说明

### `enabled`
总开关。

### `auto`
自动合成行为：
- `only_llm_result` — 只处理 LLM 的回复，忽略其他插件产生的消息
- `tts_prob` — 自动触发的概率（0-1）
- `max_msg_len` — 超过该长度的文本不自动合成
- `style_prompt` — user 消息中的默认风格指令（支持多行编辑，可写导演模式等复杂提示词）

> **去重与防护机制**：
> - 当 `/mimo说` 指令或 `mimo_tts` 工具刚调用过（15 秒内），自动模式会跳过触发，避免对同一段回复重复合成语音
> - 空文本（如工具调用后 LLM 生成的空回复）会被自动过滤，不会产生无意义音频
> - 包含图片、文件等非纯文本组件的消息会被跳过

### `client`
- `api_key` — MiMo API Key（必填）
- `base_url` — API 端点，通常无需修改
- `timeout` — 请求超时（秒）

### `model`
- `model` — 三种模型之一（默认 `mimo-v2.5-tts`）
- `voice` — 预置音色，见上方列表（默认 `mimo_default`）
- `format` — 输出格式：`mp3` / `wav` / `pcm16`（默认 `mp3`）
  - `pcm16` 为裸音频流（无文件头），插件会自动转为 WAV 格式保存与发送

### `clone`（仅 VoiceClone 模式）
- `voice_sample_path` — 参考音频文件路径（mp3/wav，数秒即可）

### `design`（仅 VoiceDesign 模式）
- `voice_design_prompt` — 文字描述用于生成全新音色

### `cache`
- `enabled` — 启用文件缓存（相同参数不再重复请求 API）
- `expire_hours` — 缓存过期时间（0 = 永不过期）
- `path` — 缓存目录（留空使用默认）

### `tool`
LLM 工具行为：
- `usage_guide` — 工具调用指引，会注入到 `mimo_tts` 工具的描述中，用于引导 LLM 何时调用、何时不调用。支持多行编辑，可自定义调用频率和场景。

`mimo_tts` 工具内置了详细的参数描述（包括风格标签格式、音频标签、唱歌模式、导演模式等示例），LLM 可以直接参考这些信息生成高质量的语音。`usage_guide` 在此基础上追加调用频率控制。

## 风格控制

V2.5 系列提供两种风格控制方式：**自然语言指令** 和 **标签控制**。

> **注意**：mimo-v2-tts（已下线）使用 `<style>开心</style>文本` 格式；V2.5 系列使用 `(开心)文本` 括号格式，两者不同。

### 1. 自然语言指令（放于 `user` 消息）

在 `style_prompt` 配置或 `mimo_tts` 工具的 `style` 参数中，直接用自然语言描述想要的语气。一段话就能生效，不需要特殊格式。

**简单示例：**
- "语气自然，像一个会聊天的私人助手"
- "慵懒随意，像在跟老朋友聊天"
- "语气尖锐刻薄，带着狐假虎威的得意感"

**导演模式（高级）：**

按「角色 → 场景 → 指导」三个维度写出更立体、更有层次的风格指令，像给演员写剧本一样：

```
角色：百年世家岑家的现任家主，自幼被宗祠守门人收养，被塑成完美无缺的情感图腾。
场景：宗祠阴影中，看着不惜冲破安保来寻她的那个人，用最冰冷最刚硬的阶级壁垒绞杀对方和自己刚刚萌生却足以燎原的感情。
指导：冰冷、慵倦却极具压迫感的成熟女声。语速极慢，每字在舌尖碾过才吐出。长停顿。全声区偏重偏硬，偶尔尾音加一点气声流露不自知的倦与渴望。
```

适用于角色配音、电影级内容生成等对声音表现力有高要求的场景。

### 2. 标签控制（放于 `assistant` 消息 / 合成文本中）

在要合成的文本开头加 `(风格)` 标签指定整体语气，也可在文本中间插入 `[音频标签]` 做细粒度控制。括号支持半角 `()`、全角 `（）`、方括号 `[]`。

**整体风格标签 — 放在文本开头：**

```text
(开心)明天就是周五了！
(悄悄话)我跟你说个秘密……
(东北话)哎呀妈呀，这事儿整的！
(粤语)呢个好犀利！食过一次就唔会忘记！
(慵懒 俏皮)让我再睡五分钟……就五分钟，真的，最后一次。
(唱歌)原谅我这一生不羁放纵爱自由，也会怕有一天会跌倒
```

风格类型参考：基础情绪（开心/悲伤/愤怒）、复合情绪（忧郁/无奈/心虚）、基调（温柔/冷漠/活泼）、音色（磁性/清亮/甜美）、角色口音（夹子音/大姐大）、方言（东北/四川/广东）、角色扮演（孙悟空/林黛玉）、唱歌

**细粒度音频标签 — 插在文本中间：**

```text
(紧张，深呼吸)好……冷静一下。(语速加快，喃喃自语)我自我介绍练了五十遍……应该没问题。(小声)啊，领带歪了吗？
(极度疲惫)师父……到了叫我一声……(长叹)我先眯一会儿。加班熬得我感觉灵魂都要飘了。
(叹气)这么多年来，再走那条街……心里突然就空了一块。(苦笑)呵，没如果了。
```

标签类型参考：语速节奏（吸气/叹气/喘气）、情绪状态（紧张/害怕/兴奋/撒娇/心虚）、发声特征（颤抖/破音/气声/嘶哑）、哭笑表达（轻笑/大笑/冷笑/抽泣/哽咽）

### 唱歌模式

仅 `mimo-v2.5-tts` 模型支持，VoiceDesign 和 VoiceClone 不支持。在文本开头加 `(唱歌)` / `(sing)` / `(singing)` 标签，后面跟歌词即可：

```text
(唱歌)原谅我这一生不羁放纵爱自由，也会怕有一天会跌倒，哦不。放弃了理想，谁人都可以，那会怕有一天只你共我。
```

歌词建议用中文，合成效果更好。

## 语音克隆用法

1. 准备一段参考音频（话术不限，几秒即可），放到插件能访问的路径
2. 设置 `clone.voice_sample_path` 为音频路径
3. 设置 `model.model` 为 `mimo-v2.5-tts-voiceclone`
4. 插件启动时将自动读取音频，转为 Base64 data URI 传入 API

**注意：** Base64 编码后不超过 10MB。

**风格控制**：VoiceClone 模式支持自然语言指令和音频标签（与 V2.5-TTS 相同的风格控制能力），但不支持唱歌模式。

## 音频格式说明

MiMo API 支持三种输出格式：

| 格式 | 特点 | 兼容性 |
|------|------|--------|
| `mp3` | 有损压缩，文件小 | 全平台通用 |
| `wav` | 无损，24kHz 16bit 单声道 | 全平台通用 |
| `pcm16` | 裸 PCM 数据流，无文件头 | **不可直接播放** |

选择 `pcm16` 时，插件会自动为音频数据添加 WAV 文件头（24kHz → 44 字节 RIFF 头），转换后的文件以 `.wav` 后缀保存，所有平台均可正常播放。

## 缓存过期机制

相同文本+相同参数的 TTS 结果会被缓存到本地，避免重复调用 API（节省费用和延迟）。

- 缓存以 SHA256 哈希命名：`mimo_<hash>.wav`
- **被动清理**：只在下次查询同一参数时检查文件创建时间，超过 `expire_hours` 自动删除并重新合成
- `expire_hours = 0` 表示永不过期
- 无后台任务，不影响插件性能
